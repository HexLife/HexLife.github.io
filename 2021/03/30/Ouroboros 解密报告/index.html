加解密逻辑

本解密工具在开发过程中参考Bitdefender的分析报告进行，报告给予了我很大的帮助，然而也发现了报告中的不足之处。报告地址：https://labs.bitdefender.com/2019/10/ouroboros-ransomware-decryption-tool/

报告中简单的介绍了一下病毒的加密模式和密钥IV信息，看似简单，实则都是最重要的信息。还有一些信息报告中没有提及，也是相当重要的信息。

在分析与编码变种1(.Lazarus)的解密工具的时候，其所使用的密钥和IV与Bitdefender的分析报告无差。具体总结如下：

  • 加密模式 ：AES - 256 - CBC - PaddingMode.Zeros

  • Key ：NC1uv734hfl8948hflgGcMaKLyWTx9H

  • IV ：1096644623466668

    病毒同样采用分块加密，分块信息如下：

  • FileSize < 1M ：病毒加密整块数据，也就是1M

  • FileSize > 1M ：病毒加密 90000 bytes 隔 810000 bytes 再加密 90000 bytes 剩余数据不加密

在分析与编码变种2(.Lazarus)的解密工具的时候，其所使用的密钥还是原来的密钥，但是IV却发生了变化，报告中只提到了一个IV，还有两个IV没在病毒的硬编码中，病毒通过计算文件大小修改了IV，具体总结如下：

  • 加密模式 ：AES - 256 - CBC - PaddingMode.Zeros
  • Key ：NC1uv734hfl8948hflgGcMaKLyWTx9H
  • IV1 ：2796949629486868
  • IV2 ：2096745621366168
  • IV3 ：1796344623568618

病毒同样采用分块加密，分块信息如下：

  • FileSize < 500K ：病毒加密整块数据，也就是500K，IV：2796949629486868
  • FileSize > 500K && FileSize < 1M ：病毒加密整块数据，也就是1M，IV：2096745621366168
  • FileSize > 1M ：病毒加密 86000 bytes 隔 814000 bytes 再加密 86000 bytes，IV：1796344623568618

按照Bitdefender的报告来解密，按道理来讲是不能够解密成功的，于是使用Bitdefender的解密工具，等了很长时间，发现该工具解密不了被加密文件。

还有一点需要说明的是加密采用的是零填充，并且被加密文件的原始大小并没有被保存下来，所以在解密时会留下一些数值为0的垃圾数据，但这并不影响文件的功能，通过测试发现大部分PE文件与dll文件的Hash和签名不会被破坏。但是一下文本文件如果用notepad++打开文件末尾会出现nul，用记事本则不会，不影响使用。但会影响部分文件的Hash。
